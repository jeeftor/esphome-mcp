     │ Context                                                                                                                                                                                                                        │
     │                                                                                                                                                                                                                                │
     │ User runs ESPHome as a Home Assistant addon. Needs an MCP server to let Claude manage ESPHome devices: read/write YAML configs, compile, install firmware OTA, pull logs, and read entity states. Two transports are relevant: │
     │ 1. ESPHome Dashboard HTTP API (port 6052) — config CRUD, compile, install, logs via SSE                                                                                                                                        │
     │ 2. ESPHome Native TCP API (port 6053, protobuf) — real-time entity states, direct device control                                                                                                                               │
     │                                                                                                                                                                                                                                │
     │ MVP priority: config management → compile/install → logs → entity states.                                                                                                                                                      │

     │ Dashboard HTTP API Endpoints Used                                                                                                                                                                                              │
     │                                                                                                                                                                                                                                │
     │ GET  /api/entries                     → list_devices                                                                                                                                                                           │
     │ GET  /api/edit?configuration=<name>  → get_config                                                                                                                                                                              │
     │ POST /api/edit  {file, content}       → save_config                                                                                                                                                                            │
     │ POST /api/validate {configuration}    → validate_config                                                                                                                                                                        │
     │ POST /api/compile {configuration}     → compile (polls /api/logs for output)                                                                                                                                                   │
     │ POST /api/upload  {configuration}     → install (OTA)                                                                                                                                                                          │
     │ GET  /api/logs?configuration=<name>  → get_logs (SSE stream, read N lines then close)                                                                                                                                          │
     │                                                                                                                                                                                                                                │
     │ Auth: Basic password via Authorization: Basic header if ESPHOME_PASSWORD is set.                                                                                                                                               │

     │ ---                                                                                                                                                                                                                            │
     │ Native TCP API (Entity States)                                                                                                                                                                                                 │
     │                                                                                                                                                                                                                                │
     │ Minimal implementation for esphome_list_entities:                                                                                                                                                                              │
     │ 1. TCP connect to host:6053                                                                                                                                                                                                    │
     │ 2. Hello handshake (protobuf HelloRequest)                                                                                                                                                                                     │
     │ 3. Send ListEntitiesRequest → collect ListEntities*Response messages                                                                                                                                                           │
     │ 4. Send SubscribeStatesRequest → collect one round of state updates                                                                                                                                                            │
     │ 5. Return combined entity+state list as JSON                                                                                                                                                                                   │
     │                                                                                                                                                                                                                                │
     │ Use esphome/aioesphomeapi's api.proto for proto definitions (copy relevant messages).                                                                                                                                          │
     │                                                                                                                                                                                                                                │

     │ Token Efficiency Design                                                                                                                                                                                                        │
     │                                                                                                                                                                                                                                │
     │ - esphome_list_devices returns minimal fields: name, state, last_seen, online                                                                                                                                                  │
     │ - esphome_get_logs returns last N lines (default 100) not a full stream                                                                                                                                                        │
     │ - esphome_compile/esphome_install return summarized output (errors + last 20 lines), not full build log                                                                                                                        │
     │ - All responses are plain text or compact JSON — no verbose wrappers                                                                                                                                                           │
